## Phase 2 Review (carried into Phase 3 prompt)

- **Built:** Command row (Views) inflated above HeliBoard's suggestion strip with five buttons (persona selector, AI toggle, Read & Respond, Sticker, AI settings). `Persona` + `FewShot` data classes serialized via `kotlinx.serialization`. `SecureStorage` wraps `EncryptedSharedPreferences` (AES-256-SIV keys / AES-256-GCM values, master key in Android Keystore) and seeds the four built-in personas (Default / Concise Editor / Esquire / Flirty) on first read. `AiSettingsActivity` (Compose, Material3, navigation-compose) launches from the gear with a persona list + edit screen. All new code lives under `com.aikeyboard.app.ai.{persona,storage,commandrow,ui}`. HeliBoard's `settings/` package and existing `SettingsActivity` are untouched.

- **Smoke test (Pixel 6 Pro `1B101FDEE0006U`, both flavors):** programmatically verified via `uiautomator dump --windows`: command row inflated for both `com.aikeyboard.app.debug` and `com.aikeyboard.app.play.debug`; persona label reads "Default" (proves `SecureStorage` seeded + read end-to-end); all five button content-descriptions present. `shared_prefs/ai_keyboard_secure.prefs.xml` exists for both flavors with binary-encrypted Tink keysets — `grep -i 'concise\|esquire\|flirty\|prompt' …prefs.xml` returns empty. Logcat clean of `FATAL` / `AndroidRuntime` referencing `com.aikeyboard.app`. play APK manifest contains zero `accessibility` references. Touchable-region fix (see Deviations) verified via `dumpsys input_method` — `touchableRegion=SkRegion((0,1624,1440,3075))` now fully includes the command row at screen y=[1769,1923]. Human-driven tap-test (gear opens settings; persona dropdown; edit + restart persistence; AI toggle visual flip) confirmed by reviewer.

- **Deviations from architecture / Phase 2 prompt:**
  - **`security-crypto:1.1.0` (stable)** instead of `1.1.0-alpha06` — `1.1.0` released since the prompt was written; lint flagged `GradleDependency`, bumped. Carries deprecation warning on `EncryptedSharedPreferences` itself (Google now steers people toward Keystore-direct or platform crypto APIs); functional and still the canonical Jetpack option, kept.
  - **`AiSettingsActivity` theme = `@style/platformActivityTheme`** (HeliBoard's `@android:style/Theme.Material.NoActionBar`) instead of `@style/Theme.AppCompat.DayNight.NoActionBar` from the prompt. Reason: HeliBoard does not depend on `androidx.appcompat:appcompat`, and the prompt forbids new dependencies beyond `security-crypto`. ComponentActivity + framework Material theme works fine with Compose; Material3 owns its own colors once `setContent` runs. Same reason for using `android.widget.PopupMenu` (not `androidx.appcompat.widget.PopupMenu`) in `CommandRowController`.
  - **No `material-icons-core` / `material-icons-extended` dep**: `Icons.Default.*` would require those libs. Followed HeliBoard's own pattern of `Icon(painterResource(R.drawable.ic_…))` and added two new vectors (`ic_ai_delete`, `ic_ai_lock`) plus reused existing `ic_arrow_back`, `ic_plus`. Five new command-row drawables also added (`ic_ai_toggle{,_on}`, `ic_read_respond`, `ic_sticker`, `ic_ai_settings_gear`).
  - **`proguard-rules.pro`** gained three `-dontwarn` lines (`com.google.errorprone.annotations.**`, `javax.annotation.**`, `javax.annotation.concurrent.**`). Tink (transitive dep of `security-crypto`) references compile-time-only annotations missing on the runtime classpath, breaking R8. The rules suppress the warnings only — no functional behavior changed. Documented inline in `proguard-rules.pro`.
  - **`ai_strings.xml`** uses `tools:ignore="MissingTranslation"` at the resources level so adding new English strings doesn't pile entries into `lint-baseline.xml` while leaving the strings translatable later (Phase 12 polish).
  - **`CommandRowView` colors come from HeliBoard's runtime `Settings.getValues().mColors`** (`ColorType.STRIP_BACKGROUND`, `TOOL_BAR_KEY`, `KEY_TEXT`) — not `?android:attr/textColorPrimary` as the first cut used. The platform-attr approach resolved against the IME's app theme, which doesn't match the active keyboard theme; icons came out near-invisible on most HeliBoard themes. The runtime-color path mirrors what `SuggestionStripView.kt` already does, so the command row visually merges with whichever theme the user has selected.
  - **`LatinIME.onComputeInsets` patched (5 lines added)**: the IME's touchable region was previously computed off `stripHeight` only (`int visibleTopY = inputHeight - visibleKeyboardView.getHeight() - stripHeight`). Our command row sits *above* HeliBoard's `strip_container`, so it fell outside the registered touchable region — taps on it were delivered to the underlying app and dismissed the keyboard. Fix subtracts the command row's height from `visibleTopY` when present. The diff in HeliBoard's territory is now: 5 lines in `setInputView`/new helper, 5 lines in `onComputeInsets`, 1 import block — minimal merge surface for upstream pulls.
  - **Long-press AND short-tap on the gear both launch `AiSettingsActivity`** (per the prompt's allowance: "short-tap also launches AiSettingsActivity for now").

- **Carried issues / chrome polish parked for later:**
  - **HeliBoard's existing toolbar coexists with our command row** — total chrome above the keys roughly doubled. PM-approved decision to defer the cleanup (hide HeliBoard's toolbar by default + add show-toggle in `AiSettingsActivity`) to Phase 12 polish for review-cleanliness reasons. Recommended approach when picking up: set `Defaults.PREF_TOOLBAR_MODE = "SUGGESTION_STRIP"` (keeps word predictions, drops HeliBoard's gear/mic/clipboard toolbar); add a "Show HeliBoard toolbar" toggle in our settings that writes `Settings.PREF_TOOLBAR_MODE`. Persistent (HeliBoard already persists to SharedPreferences). HeliBoard's full settings remain reachable through the launcher icon as escape hatch.
  - `EncryptedSharedPreferences` itself is deprecated in `security-crypto:1.1.0` (2 Kotlin warnings in `SecureStorage.kt`). Functional; replacement path TBD when Phase 3 lands API-key storage on the same lib (good moment to revisit).
  - `LocalLifecycleOwner` from `androidx.compose.ui.platform` is deprecated in favor of `androidx.lifecycle.compose.LocalLifecycleOwner` (1 warning in `PersonaListScreen.kt`); migrate when adding `androidx.lifecycle:lifecycle-runtime-compose`.
  - Compose UI uses `painterResource(R.drawable.ic_arrow_back)` from HeliBoard's existing drawables — coupling to upstream icon set. Acceptable for now; would need vendoring our own icon if upstream removes it.
  - Inherited Phase 1 issues (gesture lib not redistributable, localized HeliBoard brand strings, mipmap pruning) all still present.

- **PHASE_REVIEW.md amendment recommendation (do NOT edit yet, surface to reviewer):**
  - Future phases that add UI *above* HeliBoard's `strip_container` (Phase 3 streaming preview strip, Phase 8 kill-switch indicator) **must** either reuse the command row's host or add a corresponding height term to `LatinIME.onComputeInsets` — same pass-through bug will recur otherwise.
