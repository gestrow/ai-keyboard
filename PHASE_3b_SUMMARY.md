## Phase 3b Review (carried into Phase 4 prompt)

- **Streaming impls:** Hand-rolled SSE state machine over Ktor's `ByteReadChannel.readUTF8Line()` for both providers (no Ktor SSE plugin; reading lines is simpler and the transcript is purely line-oriented). **Anthropic** tracks the most-recent `event:` name and dispatches the matching `data:` JSON: `content_block_delta` â†’ `Delta(text)`, `message_stop` â†’ `Done`, `error` â†’ `Error` with `overloaded_error`/`rate_limit_error` mapped to `RATE_LIMITED`, auth errors mapped to `AUTH_FAILURE`. **Gemini** consumes `data:`-only chunks and walks `candidates[0].content.parts[0].text`; `finishReason` of `STOP`/`MAX_TOKENS` flushes any trailing text and emits `Done`, `SAFETY`/`RECITATION`/`BLOCKED` and `promptFeedback.blockReason` produce a single `Error` event with a generic safety-filter message. HTTP-level errors are mapped before streaming starts: 401/403â†’`AUTH_FAILURE`, 429â†’`RATE_LIMITED`, 5xxâ†’`NETWORK_FAILURE`, IO/throwâ†’`NETWORK_FAILURE`, `TimeoutCancellationException`â†’`TIMEOUT`. Body is parsed for `error.message` and surfaced verbatim when present (covers Gemini's "API key not valid" and Anthropic's "credit balance too low" with no extra translation layer needed).

- **Cancel-on-typing implementation:** single hook in `LatinIME.onEvent` that calls `mCommandRowController.cancelStreamIfActive()` *only when* `event.isFunctionalKeyEvent` is false (printable-character path). Functional events (shift, ctrl, settings) don't cancel. Three lines of HeliBoard surgery in addition to Phase 2's command-row patch and Phase 3b's preview-strip insets line.

- **Provider selection logic:** `SecureStorage.getSelectedProvider()` first tries the explicit `selectedProviderKey`; if that key was wiped or never set, falls back to `Provider.ANTHROPIC` when configured, else first non-empty key in `apiKeys`. `setSelectedProvider()` is wired but no UI surfaces it â€” Phase 6 owns the provider switcher.

- **Built:** real Anthropic + Gemini streaming in `RemoteApiBackend`; `PreviewStripView` (LinearLayout + TextView/ImageButton) hidden by default, sourcing colors from `Settings.getValues().mColors`; `LatinIME.onComputeInsets` extended one term to subtract preview-strip height when `View.VISIBLE`; rewrite icon (`ic_rewrite`) replaces the old AI toggle; `cmd_ai_toggle` renamed to `cmd_rewrite` and `setAiToggleState` / `aiModeEnabled` / the in-memory toggle state are gone; `CommandRowController` rewritten to launch a coroutine that streams into the preview strip, with selection-aware commit (selection â†’ `setSelection` + `commitText`; whole field â†’ `deleteSurroundingText` + `commitText`); 16 new JVM unit tests covering Anthropic delta/error parsing, Gemini chunk parsing (delta/done/safety/promptFeedback/malformed), and upstream-error-message extraction.

  - `KEY_HINT_LETTER` referenced in the prompt does not exist on this HeliBoard fork. Used 50% alpha of `KEY_TEXT` for the "tap to commit" hint per the prompt's fallback instruction; documented inline in `PreviewStripView.applyKeyboardTheme`.
  - Error text color: chose `Color.rgb(0xE5,0x39,0x35)` (Material A700-ish red) rather than pure `Color.RED` because pure red blends into dark themes' STRIP_BACKGROUND. Documented inline.
  - `BackendEditScreen` was using the now-deleted `ic_ai_toggle{,_on}` for its show/hide reveal toggle. Replaced with two new vector icons (`ic_visibility{,_off}`) â€” small additive change to keep the screen working after the toggle drawables were removed.
  - Provider selection: added `selectedProviderKey: String?` to `SecureData` envelope (small backwards-compatible field, defaults to null). `getSelectedProvider()` falls back to "first configured wins, default Anthropic" rule when the field is unset, matching the prompt.
  - `parseGeminiChunk` returns a `DoneWithText` variant when STOP/MAX_TOKENS chunks carry trailing text, so the final delta is flushed before `Done`. Aligns with the optional-final-delta phrasing in the prompt without complicating the streaming loop.
  - Ktor's `readUTF8Line` is deprecated (in favor of `readLineStrict`/`readLine` in Ktor 3.4+). Kept `readUTF8Line` because (a) it works correctly for SSE's LF-terminated lines, (b) the new APIs require explicit max-length params we'd need to pick, (c) compiler-warning, not lint or build error. Phase 12 polish can migrate.
