## Phase 7a Review (carried into Phase 7b prompt)

- **Service surface:** `ScreenReaderService` at `app/app/src/fdroid/java/com/aikeyboard/app/a11y/ScreenReaderService.kt`. Process-singleton handle: `@Volatile var instance: ScreenReaderService?` (private set), assigned in `onServiceConnected`, cleared in BOTH `onUnbind` AND `onDestroy` (defense against lifecycle paths that skip `onUnbind`). `requestScreenContext(): ScreenReaderResult` walks `rootInActiveWindow` once, returns `Success(ScreenContext)` or a typed `Failure` (`SERVICE_NOT_ENABLED` / `NO_ACTIVE_WINDOW` / `BUILD_DOES_NOT_SUPPORT` / `UNKNOWN_FAILURE`). DFS traversal capped at `MAX_NODES = 2_000`; outer try/catch over `Throwable` also catches `StackOverflowError` (graceful `UNKNOWN_FAILURE` return). Synchronous, main-thread call. **Why singleton, not bound service:** `AccessibilityService.onBind()` is `final` in the platform — overriding it doesn't compile, so the standard `bindService()` IBinder dance is unavailable. `PHASE_REVIEW.md:302` explicitly permits the singleton alternative; IME and a11y service share the same app process by default, so direct reference is safe. No AIDL or `Parcelable`.

- **Service config (`app/app/src/fdroid/res/xml/accessibility_service_config.xml`):** `accessibilityEventTypes=""` (empty mask — zero events subscribed; AAPT2 rejects `"0"` for this flags attribute, the empty string is the legal "no flags" form), `canRetrieveWindowContent="true"`, `accessibilityFeedbackType="feedbackGeneric"`, `notificationTimeout="100"`, `accessibilityFlags="flagDefault|flagRetrieveInteractiveWindows"` (the second flag lets `getRootInActiveWindow()` include floating/dialog windows). Description string is explicit about press-to-read-once and "no data leaves your device beyond what you would normally send to your selected AI backend."

- **ScreenContext API (in `app/app/src/main/java/com/aikeyboard/app/ai/a11y/`):** `ScreenContext(nodes, focusedInputIndex, aboveInputText, focusedInputText, nodeCount, walkDurationMs)`; `TextNode(text, isInputField, isFocused, className)`; `sealed interface ScreenReaderResult { Success(context); enum Failure { SERVICE_NOT_ENABLED, NO_ACTIVE_WINDOW, BUILD_DOES_NOT_SUPPORT, UNKNOWN_FAILURE } }`. Lives in `main`, not `fdroid`, so Phase 7b's wiring code (compiled into both flavors, gated by `BuildConfig.ENABLE_A11Y`) can reference the type without flavor-conditional imports. `text` derivation rule: `node.text ?: node.contentDescription ?: ""` — does NOT fall back to `hintText` (would reveal "Enter password" / "Credit card number" even on empty fields). `aboveInputText` joins `nodes[0..focusedInputIndex)` with `\n`; empty when `focusedInputIndex <= 0`. `focusedInputText` is `nodes[focusedInputIndex].text` or `""`.

- **Deviations from Phase 7a prompt:**
  - `accessibilityEventTypes=""` (literal empty string) chosen over `"0"`. AAPT2 rejects `"0"` for the flags attribute (`'0' is incompatible with attribute accessibilityEventTypes (attr) flags`). The prompt's task description and verification both name `""`; this is the only form that compiles. Functionally identical (both express "no flags").
